# 해석 의뢰 현황판 (Analysis Request System)

본 저장소는 가상검증팀의 **해석 의뢰 현황판**을 함께 개발하기 위한 협업 공간입니다. 기존 **작업의뢰 시스템**을 기반으로 하면서도 해석 업무에 특화된 기능을 추가하여 사용자가 의뢰, 승인, 수행, 완료까지 모든 과정을 효율적으로 관리할 수 있도록 돕습니다. 이 프로젝트는 [MS]개발 프로젝트에서의 개발 방식과 동일하게 진행되며, Power Apps와 SharePoint 리스트를 활용하는 것을 전제로 합니다.

## 주요 기능 개요

1. **의뢰 요청 및 승인 절차**
   - 의뢰자가 해석 의뢰를 제출하면 팀장 승인 과정을 거칩니다.
   - 승인 요청은 가상검증팀 팀장 및 최고 선임자에게 이메일로 자동 발송됩니다.
   - 가상검증팀에서 승인 시 담당자가 배정되고, 담당자는 이메일을 통해 접수 알림을 받습니다.

2. **수정‧보완 기록 관리**
   - 의뢰 내용은 최대 15차례까지 수정할 수 있으며, 각 수정 본(A, B, C…)의 내용과 첨부파일을 모두 유지합니다.
   - 의뢰자가 최종 버전(C 등)을 보더라도 이전 기록을 리스트로 확인하고 선택하여 상세내용과 첨부파일을 열람할 수 있습니다.

3. **진행 상태 공유**
   - 작업자가 진행률(%)과 예상 완료일을 직접 입력할 수 있는 UI를 제공합니다.
   - 의뢰자는 따로 문의하지 않고도 진행 상태를 확인할 수 있습니다.

4. **작업 중 수정 사항 기록 및 첨부파일 업로드**
   - 작업자는 해석 과정에서 수정한 내용을 텍스트로 입력하고 관련 자료를 첨부할 수 있는 섹션을 사용합니다.

5. **의뢰 완료 및 해석보고서**
   - 작업자가 모든 작업을 완료하면 해석 보고서를 첨부해야만 완료처리가 가능합니다.
   - 보고서 첨부 후 의뢰자에게 완료 알림이 전송됩니다.

6. **조회 및 필터링**
   - 사용자는 전체 의뢰와 본인 의뢰만을 토글로 전환할 수 있습니다.
   - 진행 중, 완료 등 단계별 조회가 가능하며, 인쇄(프린트)용 버튼을 제공합니다.

## 구조 및 파일

현재는 기획 및 설계 단계이기 때문에 기본적인 레포지토리 구조만 마련해 두었습니다. 실제 코드와 스크린 정의(YAML 또는 파워앱 .msapp 파일)는 향후 개발 과정에서 추가될 예정입니다.

```
analysis-request-system/
├── powerapps/        # Power Apps 화면 정의(YAML 또는 .msapp)를 위한 디렉터리
├── docs/             # 요구사항 명세, 설계 문서 등을 저장
├── .gitignore        # Git에서 추적하지 않을 파일 목록
├── CONTRIBUTING.md   # 협업 규칙과 코드 작성 지침
└── README.md         # 프로젝트 소개와 사용 안내
```

> **참고:** 작업의뢰 시스템에서 가져온 YAML 예제는 `powerapps/AnalysisRequest_Screen_fullcode.yaml`에 임시로 포함되어 있습니다. 이는 참고용이며 실제 개발 과정에서 수정되거나 교체될 수 있습니다.

## 사용 및 개발 방법

1. **요구사항 파악:** 이 README에 요약된 기능을 기반으로 추가 요구사항이나 세부 규칙을 논의하고 문서화합니다. `/docs` 폴더를 활용해 명세서를 작성하세요.
2. **데이터 모델 설계:** SharePoint 리스트 및 컬럼을 설계합니다. 기본 필드는 작업의뢰 시스템과 동일하지만, 아래와 같은 컬럼을 추가합니다:
   - `RevisionHistory` ‒ 각 수정 기록을 저장할 별도 리스트와 연계하거나 JSON 형태로 저장.
   - `Progress` ‒ 진행률(0~100). 진행상황 공유용.
   - `ExpectedCompletionDate` ‒ 예상 완료일(날짜 형식).
   - `ReportAttachment` ‒ 해석 완료 시 업로드되는 보고서 파일.
3. **Power Apps 개발:** `powerapps` 디렉터리에 화면 정의 파일을 추가합니다. 작업의뢰 시스템의 코드를 참고하여 60%는 재사용하고, 요구사항에 맞게 수정합니다.
4. **공동 개발:** Pull Request 기반 협업을 권장합니다. 변경 사항은 모두 코드 리뷰를 거쳐 반영합니다.
5. **배포:** 완성된 앱을 Teams 채널에 게시하여 사용자가 손쉽게 접근할 수 있도록 합니다.

## 개발 규칙 및 참고 사항

* 이 프로젝트는 내부 연구소 용도로 설계되어 외부에 공개하지 않습니다. 코드 및 데이터 처리 시 회사 정책과 보안 규정을 준수해야 합니다.
* 파워앱 무료 라이선스 범위에서 개발되므로, Power Automate나 Dataverse 등의 유료 기능 대신 SharePoint 및 Excel을 활용하세요.
* Pull Request에는 변경된 기능, 수정 이유 등을 명확히 기재해 주세요.
